# Prompt used to create this tool

このリポジトリのスクリプトは、以下のような要件をもとに、生成AIとの対話で作成・調整したものです。

同じようなツールを作りたい場合や、仕様を見直す場合の参考として、最終的な要件をプロンプト形式で残しています。

---

## Prompt

Windows環境で動作するPowerShellスクリプトと、それをドラッグ＆ドロップで起動するためのBATファイルを作成してください。

目的は、e-Govで電子送達される社会保険関連のお知らせZIPファイルを整理することです。

e-GovからダウンロードされるZIPファイルは、機械的な名前になっており、そのままでは内容や年月が分かりにくいため、ZIP内のXMLファイルを調べて、通知の種類と年月に応じた分かりやすいフォルダ名で展開したいです。

## 作成するファイル

以下の2ファイルを作成してください。

* `Extract-JPSZipFolders.ps1`
* `Extract-JPSZipFolders.bat`

`Extract-JPSZipFolders.bat` にZIPファイルをドラッグ＆ドロップすると、同じフォルダにある `Extract-JPSZipFolders.ps1` を呼び出して処理する構成にしてください。

複数のZIPファイルをまとめてBATにドラッグ＆ドロップした場合は、1つずつ処理してください。

## 基本動作

ZIPファイルの中を再帰的に調べ、ZIP内の階層にあるXMLファイルを探してください。

該当するXMLファイルが見つかった場合、そのXMLファイルが存在するZIP内フォルダを、通知の種類と年月に応じた名前で展開してください。

ZIP全体を無条件に展開するのではなく、該当XMLファイルがある上位フォルダ単位で展開してください。

展開する前に、以下を表示してください。

* 判定したパターン
* ZIP内の対象フォルダ
* 優先展開先
* 実際の展開先
* 展開先フォルダ名

そのうえで、展開してよいかどうかをユーザーに確認してください。

`y`, `Y`, `yes`, `YES` の場合のみ展開し、それ以外はスキップしてください。

## 展開先

通知の種類ごとに、優先展開先フォルダをスクリプト冒頭で設定できるようにしてください。

例：

```powershell
$OutputPathPattern1 = "\\server\share\年金\被保険者データ"
$OutputPathPattern2 = "\\server\share\年金\納入告知額・領収済額通知"
$OutputPathPattern3 = "\\server\share\年金\社会保険料額情報"
```

指定した優先展開先フォルダが存在し、かつ書き込み可能であれば、そこへ展開してください。

優先展開先フォルダが存在しない場合、ネットワークパスに到達できない場合、または書き込みできない場合は、エラーで停止せず、Desktopへ展開してください。

ネットワークパスに到達できない場合でも、`Test-Path` のエラーでスクリプト全体が停止しないようにしてください。`Test-Path` は `try/catch` で囲むだけでなく、`-ErrorAction Stop` を付けて、ネットワークパス不達時のエラーを確実に捕捉してください。捕捉した場合は、優先展開先は使えないものとして扱い、Desktopへフォールバックしてください。

書き込み可能かどうかは、一時ファイルを作成して削除できるかで確認してください。

## 展開後の動作

1件でも展開に成功した場合、最後に、展開したフォルダそのものではなく、その1つ上の親フォルダをExplorerで開いてください。

複数の親フォルダに展開した場合は、それぞれの親フォルダを1回ずつExplorerで開いてください。

## フォルダ名の重複対応

展開先に同名フォルダがすでに存在する場合は、上書きせず、以下のように連番を付けてください。

```text
令和8年3月分社会保険料額情報
令和8年3月分社会保険料額情報 (2)
令和8年3月分社会保険料額情報 (3)
```

## フォルダ名の安全化

Windowsのフォルダ名として使えない文字は除去してください。

対象文字は以下です。

```text
\ / : * ? " < > |
```

## 対象パターン

以下の3パターンを判定してください。

判定順は、パターン2、パターン3、パターン1の順にしてください。

理由は、パターン2・3にもXMLファイルが存在するため、先にパターン1を判定すると誤判定する可能性があるためです。

## パターン1: 被保険者データ

同一ZIP内フォルダに、以下の3ファイルがある場合を対象にしてください。

* `*.xml`
* `*.xsl`
* `*.DTA`

ただし、XMLファイルが複数存在する場合は、メッセージを表示して、そのフォルダの処理はスキップしてください。

XMLファイルを読み取り、以下のような構造を想定してください。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="kagami.xsl" ?>
<DOC>
    <BODY ID="DOCBODY">
        <APPENDIX>
            <APPTITLE>被保険者データ_令和07年10月分(XXXXXXXX.DTA)※令和7年10月23日までに入力処理した情報を収録</APPTITLE>
        </APPENDIX>
    </BODY>
</DOC>
```

`DOC > BODY > APPENDIX > APPTITLE` の文字列から、`被保険者データ_` の後から `(` の間の部分を抜き出してください。

上記例では、以下を抜き出します。

```text
令和07年10月分
```

展開先フォルダ名は、以下の形式にしてください。

```text
令和07年10月分被保険者データ
```

## パターン2: 納入告知額・領収済額通知

ZIP内フォルダに、以下で始まるXMLファイルが1つある場合を対象にしてください。

```text
保険料納入告知額・領収済額通知書_*.xml
```

例：

```text
保険料納入告知額・領収済額通知書_令和7年12月送付分(202512000000000000).xml
```

ファイル名のうち、`保険料納入告知額・領収済額通知書_` の後から、`(` の手前までを抜き出してください。

上記例では、以下を抜き出します。

```text
令和7年12月送付分
```

展開先フォルダ名は、以下の形式にしてください。

```text
令和7年12月送付分納入告知額・領収済額通知
```

該当XMLファイルが複数存在する場合は、メッセージを表示して、そのフォルダの処理はスキップしてください。

上記例では、以下を抜き出します。


展開先フォルダ名は、以下の形式にしてください。

```text
令和8年3月分社会保険料額情報
```

## エラー処理

以下の場合は、スクリプト全体を停止せず、該当フォルダだけスキップしてください。

* XMLファイルが複数ある
* XMLを読み取れない
* 必要なXML要素が見つからない
* ファイル名から年月部分を取得できない
* 展開先フォルダ名が空になる
* 展開に失敗する

ただし、指定されたZIPファイル自体が存在しない場合や、ZIPファイルではない場合は、エラーメッセージを表示して終了してください。

## 文字コード

日本語のファイル名や日本語文字列を扱うため、PowerShellスクリプトはWindows PowerShell 5.1でも文字化けしにくいよう、UTF-8 with BOMで保存する前提にしてください。

BATファイルは文字化けを避けるため、表示メッセージはできるだけASCII中心にしてください。

## BATファイル

BATファイルは、以下の役割にしてください。

* ドラッグ＆ドロップされたZIPファイルを受け取る
* 同じフォルダにあるPowerShellスクリプトを呼び出す
* 複数ファイルが渡された場合は1つずつ処理する
* 最後に `pause` して結果を確認できるようにする

PowerShell呼び出し時は、以下のように実行ポリシーを一時的に回避してください。

```bat
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "%SCRIPT%" "%~1"
```
